ТЗ ДЛЯ CODEX: Web-app “Seed-like” (для мене, локально/без акаунтів)

1. ПРИКЛАД ТИ МОЖЕШ ПОБАЧИТИ У ФАЙЛІ "Приклад"

1) Ціль
Зробити простий сайт (PWA необов’язково, але бажано), який допомагає мені:
- вести streak (дні без порно / без зриву)
- отримувати level up за прогрес
- бачити “orb/seed” який еволюціонує з рівнями (проста 2D-графіка, SVG/Canvas)
- мати кнопку “Мене тригерить” з короткими вправами (breathwork/таймер/нагадування)
- переглядати статистику (календар, графік)

2) Технології (обрати і реалізувати)
- Next.js (App Router) + TypeScript + Tailwind
- Зберігання: localStorage (MVP). Додай експорт/імпорт JSON.
- Деплой: Vercel (але має працювати локально через npm run dev)

3) Основні сторінки
A) / (Dashboard)
- Великий блок: “Streak: X днів”
- Поточний Level: N + прогрес-бар до наступного рівня
- Візуал “Orb” (SVG/Canvas), який змінюється по рівнях (мінімум 6 стадій)
- Кнопки:
  1) “✅ Check-in сьогодні (без зриву)”
  2) “⚠️ Зрив сьогодні”
  3) “🔥 Мене тригерить” (відкриває Anti-Urge modal)
- Малий блок мотивації: випадкова фраза/нагорода дня

B) /stats
- Календар (місяць) з позначками:
  - зелений: successful day
  - червоний: relapse
  - сірий: no data
- Графік за 30 днів: streak trend / кількість успішних днів
- Лічильники:
  - total clean days
  - total relapses
  - best streak
  - current streak

C) /settings
- Вибір режиму:
  - “No Porn”
  - “No PMO”
  - “Reduce frequency” (для цього режиму дозволяється встановити max разів на тиждень; MVP можна залишити як TODO)
- Налаштування нагадувань (тільки якщо PWA/Notifications, інакше просто UI-заглушка):
  - час чек-іну
- Кнопки:
  - Export data (download JSON)
  - Import data (upload JSON)
  - Reset all data (з підтвердженням)

4) Логіка streak і level
- Дані по днях зберігаються як записи DailyLog:
  - date (YYYY-MM-DD)
  - status: “clean” | “relapse”
  - note (optional)
- Current streak:
  - рахується як кількість останніх послідовних днів “clean”, включно з сьогоднішнім якщо clean.
- Best streak:
  - максимум по історії.
- Level:
  - level = floor(totalCleanDays / 7) + 1 (тобто кожні 7 чистих днів +1 рівень)
  - progressToNext = totalCleanDays % 7 (0..6)
- Orb evolution:
  - 6 стадій за рівнями:
    stage 1: levels 1-2
    stage 2: levels 3-4
    stage 3: levels 5-6
    stage 4: levels 7-9
    stage 5: levels 10-12
    stage 6: levels 13+
  - Кожна стадія має різний вигляд (форма/патерн/світіння) у SVG або Canvas.
  - Не використовуй зовнішні картинки: намалюй сам через SVG.

5) Anti-Urge modal (“Мене тригерить”)
Вікно з 3 швидкими діями:
- “Дихання 4-7-8” (таймер + підказка, 4 цикли)
- “Таймер 10 хв” (countdown з текстом: ‘потяг пройде’)
- “Швидкий план” (3 кроки):
  1) Відійди від телефону/ПК на 2 хв
  2) 20 віджимань або 1 хв планки
  3) Випий воду + відкрий Dashboard
Також кнопка “Додати нотатку, що мене тригернуло” (зберегти note у сьогоднішній запис або в окремий TriggerLog)

6) UI/UX вимоги
- Мінімалістично, темна і світла тема (toggle)
- Адаптивність під телефон
- Анімації легкі: прогрес-бар, orb transition, confetti при level up (можна простий CSS/Canvas confetti)

7) Структура проекту
- components/: Orb, ProgressBar, Calendar, AntiUrgeModal, StatsCards
- lib/: streak.ts (обчислення), storage.ts (localStorage), dates.ts
- types/: models.ts

8) Acceptance criteria (щоб вважати готовим)
- Можу відмітити “clean” або “relapse” за сьогодні, і це коректно впливає на streak.
- Level і прогрес-бар рахуються коректно.
- Orb змінюється по рівнях (мінімум 6 стадій).
- Статистика показує календар і основні числа.
- Дані не губляться після перезавантаження сторінки (localStorage).
- Export/Import працює (JSON).
